# 约发 YueFa - MVP精简版PRD

**核心逻辑：** 对外做"防屏蔽的超级名片"，对内做"简单的订单管理"

---

## 1. 产品定位

一款帮助Cosplay假发造型师（毛娘）**展示作品+收集询价+管理订单**的移动端H5工具。

**MVP目标：** 让毛娘能发链接，客户能填表，毛娘能管理订单全流程。

---

## 2. MVP功能范围

### ✅ 保留（P0核心功能）

| 功能 | 描述 |
|------|------|
| **作品集展示** | 瀑布流展示作品图片（暂不支持视频），支持标签筛选 |
| **询价表单** | 客户填写：角色名、作品名、期望工期、头围、预算范围、人设图上传 |
| **联系方式** | 显示毛娘微信号，支持一键复制 |
| **询价列表** | 毛娘端查看所有询价请求 |
| **订单管理** | 简单列表+状态切换（新询价→待定金→制作中→待发货→已完成） |
| **订单详情** | 查看客户信息、人设图、制作要求、可追加备注 |

### ❌ 砍掉（MVP后再做）

| 功能 | 砍掉原因 |
|------|----------|
| 看板拖拽/Kanban | 复杂度高，列表+下拉切换状态足够 |
| 智能日历 | 非核心，手机日历可替代 |
| 协议签署功能 | 可在微信沟通时确认 |
| 档期红绿灯 | 可在公告区手动写明 |
| 一键生成回执 | 非核心，截图即可 |
| 客户黑名单库 | 后期功能 |
| 个性化皮肤 | 商业化功能 |
| 会员订阅系统 | 先免费跑通流程 |
| 视频作品集 | 技术复杂度高，先做图片 |
| 域名池自动切换 | MVP先用单域名测试市场 |

---

## 3. MVP用户流程

```
【配置阶段】
毛娘注册 → 上传作品+设置微信号 → 获得专属链接

【获客阶段】
客户点击链接 → 浏览作品 → 填写询价表单 → 提交

【管理阶段】
毛娘收到通知 → 查看询价 → 微信沟通确认 → 转为订单 → 更新状态直至完成
```

**订单状态流：**
```
新询价 → 待定金 → 制作中 → 待发货 → 已完成
```

---

## 4. MVP技术方案

| 端 | 技术选型 | 说明 |
|----|----------|------|
| 客户端 | H5网页 | 无需下载，链接即用 |
| 毛娘端 | H5网页 | 暂不做小程序，降低开发成本 |
| 后端 | Serverless | 快速上线，按量付费 |
| 数据库 | 云数据库 | 简单存储即可 |

---

## 5. MVP验证指标

1. **核心假设：** 毛娘愿意用这个工具替代直接发微信号
2. **验证方法：** 找10个毛娘试用，观察是否持续使用
3. **成功标准：** 7天后仍有50%+在使用

---

## 6. MVP开发范围（5个页面）

1. **毛娘注册/登录页** - 手机号验证码登录
2. **毛娘主页编辑页** - 上传头像、作品图、设置微信号、公告
3. **客户浏览页（对外）** - 展示作品+询价表单
4. **订单列表页（毛娘端）** - 查看所有订单，支持按状态筛选，下拉切换状态
5. **订单详情页（毛娘端）** - 查看客户信息、人设图、添加备注

---

## 7. 暂缓考虑的问题

- 域名被封怎么办 → 先跑通流程，被封再说
- 图片加载慢怎么办 → 先用CDN，后续优化
- 需要订单管理吗 → MVP只做到询价收集，订单管理用微信+Excel

---

**MVP核心原则：能省则省，能砍则砍，先让用户用起来再说。**
