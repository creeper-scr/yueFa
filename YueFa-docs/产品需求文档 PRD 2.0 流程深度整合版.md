**项目名称：** 约发 YueFa
**版本号：** v1.0 (MVP)  
**核心理念：** 还原真实接单链路，把控关键节点的风险（定金、毛坯、验收）。

---

## 1. 产品逻辑架构 (The Logic)

我们将毛娘的业务流抽象为 **“5阶段状态机”**，软件的所有功能围绕订单状态的流转设计。

- **S1 询单期：** 需求沟通 -> 档期匹配 -> 报价
    
- **S2 准备期：** 支付定金 (20%) -> **毛坯到货管理** (新增关键点)
    
- **S3 制作期：** 排单制作 -> **数字化验收/修整**
    
- **S4 交付期：** 支付尾款 (80%) -> 包装发货
    
- **S5 售后/归档：** 确认收货 -> 评价/纠纷处理
    

---

## 2. 核心用户流程 (User Journey)

### 2.1 理想路径

1. **[客户]** 访问H5主页 -> 填写《定制需求单》（含头围、毛坯来源） -> 提交。
    
2. **[毛娘]** 收到通知 -> 检查档期 -> 填写报价（系统自动算定金） -> 生成《接单确认函》。
    
3. **[客户]** 确认报价 -> 支付20%定金（私下转账，上传截图） -> **寄出毛坯/支付代购费**。
    
4. **[毛娘]** 确认定金到账 -> **确认收到毛坯** -> 状态变更为“待排单”。
    
5. **[毛娘]** 制作完成 -> 上传360°视频/图片 -> 生成“验收链接”。
    
6. **[客户]** 点击链接查看 -> 点击“确认满意”（或“申请修改”） -> 支付80%尾款。
    
7. **[毛娘]** 确认尾款 -> 发货并录入单号 -> 交易结束。

## 3. 功能模块详解 (Functional Requirements)

### 模块一：智能询单与报价 (Inquiry & Quote)

**对应流程：第一阶段（咨询） & 第二阶段（定金）**

|   |   |   |   |
|---|---|---|---|
|功能ID|功能名称|需求描述|业务价值|
|**F-01**|**结构化需求表单**|客户必填字段：<br>1. **角色/作品**<br>2. **期望工期** (关联日历)<br>3. **头围数据** (提供测量示意图，不仅填数字，建议填备注)<br>4. **特殊要求** (发际线/炸毛/鬓角)<br>5. **毛坯来源** (选项：我寄给你 / 你帮我买)|避免反复询问，防止漏记头围导致售后。|
|**F-02**|**自动报价计算器**|毛娘输入总价（如500元），系统自动拆分：<br>- **定金 (20%)：** 100元 (提示：跑单不退)<br>- **尾款 (80%)：** 400元|规范行业定金比例，减少口算错误。|
|**F-03**|**毛坯物流追踪**|若客户选“我寄给你”，生成一个**“待收货”**卡片。<br>毛娘收到快递后，点击“确认收货”，订单才正式进入排单池。|**痛点解决：** 很多时候定金付了，毛坯半个月没寄来，导致毛娘空档。此功能倒逼客户发货。|

**对应流程：第三阶段（制作）**

**对应流程：第三阶段（验收） & 第四阶段（尾款）**

|   |   |   |   |
|---|---|---|---|
|功能ID|功能名称|需求描述|业务价值|
|**R-01**|**验收专用页**|毛娘上传成品视频/高清图，生成唯一链接发给客户。<br>页面包含两个大按钮：<br>1. **✅ 确认满意，去付尾款**<br>2. **🛠 申请修改 (剩余次数: 2)**|将“验收”环节仪式化，保留证据。限制修改次数，防止无休止返工。|

## 4. 关键字段定义 (Data Fields)

为了支撑上述流程，数据库需重点记录以下字段：

- **Order_Status (枚举值):**
    
    - PENDING_QUOTE (待报价)
        
    - PENDING_DEPOSIT (待付定金)
        
    - AWAITING_WIG_BASE (定金已付，等毛坯) -> **这是区分小白和专业SaaS的关键点**
        
    - QUEUED (排单中)
        
    - IN_PROGRESS (制作中)
        
    - IN_REVIEW (验收中/修改中)
        
    - PENDING_BALANCE (验收通过，待尾款)
        
    - SHIPPED (已发货)
        
    - COMPLETED (已完成)
        
- **Wig_Source (枚举值):**
    
    - CLIENT_SENDS (客户寄来)
        
    - STYLIST_BUYS (毛娘代购)

## 5. UI/UX 设计重点 (Design Highlights)

1. **移动端优先：** 所有操作（包括拖拽看板）必须适配手机竖屏。
    
2. **毛坯状态显性化：** 在看板上，如果是“客户寄毛坯”且未收到，卡片上显示一个小快递箱图标📦（灰色）；收到并确认后，变亮（绿色）。
    
3. **验收页面的“仪式感”：** 客户打开验收链接时，背景要是干净的，图片支持放大查看细节，确认按钮要做得大而令人愉悦（给予心理暗示：这顶假发很棒）。

## 6. MVP 开发优先级排序

1. **P0 (必须有):**
    
    - H5询价单（含头围、毛坯来源）。
        
    - 后端看板（含“待收毛坯”状态）。
        
    - 简单的验收链接（上传图片 -> 客户看 -> 口头确认后毛娘手动改状态）。
        
2. **P1 (下一步):**
    
    - 验收页面的交互功能（点击确认/修改）。
        
    - 自动算出20%/80%金额。
        
    - 发货Checklist。
        
3. **P2 (未来):**
    
    - 导出PDF维权凭证。
        
    - 物流接口对接（自动查询毛坯快递到了没）。

## 7. 总结

这款SaaS与通用To-do软件最大的区别在于它**深刻理解“毛坯”和“验收”这两个行业特有的痛点**。

- 通用软件不知道“定金付了但假发没寄来”该怎么算工期。
    
- 通用软件没有“看图-反馈-修改-再看图-确认-付尾款”的闭环逻辑。  
    **抓住这两点，就是抓住了毛娘工作流的命门。**
